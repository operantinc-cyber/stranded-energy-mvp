# V2 Roadmap

This roadmap focuses on practical improvements that make the MVP easier to demo, validate, and extend.

## Demo And Export

- Add PDF export for project memos.
- Add data room export with memo, risk register, assumptions, and result tables.
- Add a demo data reset command.
- Add cleaner Markdown formatting for exported memos.

## Analysis Improvements

- Add scenario comparison for scoring, gas-to-power, and financial screens.
- Add side-by-side saved cases.
- Add richer sensitivity tables.
- Add advanced financial model with debt, tax, depreciation, IRR, NPV, construction timing, and escalation.
- Add shared formatting helpers for money, MW, MWh, dates, percentages, and classifications.

## Quality And Testing

- Add better unit tests for:
  - scoring
  - gas-to-power
  - financial screen
  - risk severity
  - monetization ranking
  - memo generation
- Add workflow tests for the end-to-end opportunity path.
- Add clearer form validation and friendly error messages.

## Workflow

- Add opportunity pipeline Kanban.
- Add stage/status management.
- Add task ownership and due dates.
- Add memo version history.
- Add activity log for saved calculations and changes.

## Documents And CRM

- Add document upload.
- Add document tagging by opportunity.
- Add AI-assisted memo drafting with explicit source references.
- Add counterparty CRM for owners, operators, offtakers, vendors, lenders, and advisors.
- Add contact history and next-action tracking.

## Mapping And Data

- Add GIS/map view.
- Add manual map pins.
- Add optional external data integrations for pipelines, substations, power lines, roads, fiber, loads, and data centers.
- Add structured data import from spreadsheets.
- Add automated external data ingestion where appropriate.

## Productization

- Add authentication and user roles.
- Move from local SQLite to a production database.
- Add deployment documentation.
- Add backups.
- Add audit trail.
- Add organization/workspace support.

## Messaging And Company Narrative

- Keep the company positioned as an AI-native energy infrastructure developer.
- Keep the current MVP positioned as a concierge development screen.
- Keep autonomy language tied to project origination, screening, and development coordination.
- Avoid describing the business as a generic SaaS product or dashboard company.
